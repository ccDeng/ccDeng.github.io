实现记住密码与登录超时功能

发表于 2016-03-25   |   分类于 js   |  

有关如何阻止浏览器记住你的密码，以及实现登陆超时自动注销功能。

【一】 关于记住密码状态

  • 记住密码采取的方法可以存储在本地的local storage中 或者在 cookie中，两者区别是local storage安全性相比cookie要差，而cookie的存储大小则比local storage要小。

  • 一般的处理方法是在打开登陆页面时候检查本地有无储存数据，将数据填充进去。但是这些都是在js中操作，而浏览器是可以记住你输入的表单密码数据的，它会弹出来提示小窗口问你是否记住密码，而这个密码我们是可以在浏览器设置中找到的，并且它会自动在你打开登陆页的时候填充进去。

  • 那么我不想要记住密码时候应该怎么操作：

    1. autocomplete=”off”属性

    2. 页面采用https协议，因为https是不会对表单进行保存信息的

    3. 采用flash做登录表单，这种情况浏览器也不会记录表单信息

    4. 浏览器实际是根据type=”password”来获取记得表单密码信息的，动态设置密码域

  • 那然后我们接下来谈一下，这几个方法的具体实现，以及利弊问题

    1. autocomplete属性是不要自动填充，但是这个方法再chrome中是没有用的，所以可以在写代码时候以防万一写上去，对于firefox等浏览器还是有用的。

    2. 至于页面采用https协议，这是项目开始之前就定好的，所以这个方法不能在项目进行时采用

    3. 采用flash做表单，但是因为涉及网站优化问题所以并不推荐

【题外话】关于session的了解

  1. session代表服务器与浏览器的一次会话过程，这个过程是连续的，也可以时断时续的，但是它的具体含义有很多的扩展

  2. session机制采用的是在服务器端保持状态的方案，举例来说，就是每个人都有学号，但是我们的学号并没有信息，你要查询学生的信息时候，提供学生学号，学校可以从档案中查询你的信息，这就是在服务器端保存。

  3. cookie的对比，cookie机制采用的是在客户端保持状态的方案，举例来说，也就是每个学生有一张学生卡，这张卡里面就已经记录了他的信息。

  4. 由于采用服务器端保持状态的方案在客户端也需要保存一个标识，所以session机制可能需要借助于cookie机制来达到保存标识的目的，但是并不止这一种方法。（如JSESSIONID）
