关于HTTP

发表于 2016-05-20   |   分类于 HTTP   |  

与HTTP相关。

  • 首先，要知道HTTP是什么。
    HTTP是Hyper Text Transfer Protocol的缩写，也称超文本传输协议，是用于从万维网服务器传输超文本到本地浏览器的传送协议。其主要特点是：支持C/S模式，无连接，无状态，快速灵活。

    HTTP的工作原理为：
    1.客户端TCP链接到服务器
    2.客户端发送请求
    3.服务器响应
    4.释放TCP
    5.客户端解析

接下来，了解一些更具体的东西

  • HTTP的URL
    首先说明URI跟URL。
    HTTP采用统一资源标识符（即URI）来传输数据和建立连接，而（统一资源定位符）URL包含在URI中，两者最明显的区别URL指明了如何locate到这个资源，在IETF中将URL定义为URI的子集，举例来说，人的身份证号就是一个URI，但是这个号码是locate不到这个人的。
    HTTP中URL的格式为 http: //host[:port][abs_path]
    其中http是协议，host为主机地址，port为端口号，abs_path为请求资源的URI

  • HTTP的请求（request）与响应（response）
    HTTP的request报文由三部分组成，包括请求行、首部行、实体主体。其中请求行包括方法、URL、HTTP版本。
    response报文由三部分组成，包括状态行、首部行、实体主体。其中状态行包括HTTP版本、状态码、简单短语。

  • HTTP请求方法
    OPTION： 请求一些选项的信息
    GET：请求读取URL标志的信息
    HEAD： 请求读取URL标志的信息首部
    POST： 给服务器添加信息（GET跟POST请求的区别在于GET传参数附在URL后，POST传Params单独存在；GET请求有长度限制，POST没有；POST安全性要高于GET，不缓存在浏览器中，也不显示在url中）
    PUT： 在URL下存储文档
    DELETE： 删除URL下资源
    TRACE：进行环回测试
    CONNECT： 代理服务器

  • HTTP状态码
    1xx： 表示通知信息，如请求收到了或正在处理
    2xx： 表示成功，如接受或者知道
    3xx： 表示重定向，如完成请求还需进一步
    4xx： 表示客户端的错误，不能完成
    5xx： 服务器错误，不能完成

  • 服务器存放信息（Cookie）
    我们上面已经说过，HTTP是无状态的，但是在实际中很多网站需要识别用户。客户端在访问服务器时，服务器会产生一个识别码，然后响应报文会添加Set-cookie的首部，然后之后每个请求报文都会有Cookie发送过去服务器。
